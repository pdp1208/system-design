LinkedIn - Flow Chart

Flow Chart

Visual representation of LinkedIn's key workflows and user interaction flows.

1. User Registration & Onboarding Flow

User lands on LinkedIn
↓
Sign Up Form (Email/Phone)
↓
Email/SMS Verification
↓
Basic Profile Creation (Name, Photo, Headline)
↓
Add Work Experience
↓
Add Education
↓
Connect with People (Import contacts, Search)
↓
Follow Companies & Influencers
↓
Complete Profile (Skills, Recommendations)
↓
Home Feed Ready

2. User Authentication Flow

User enters credentials
↓
Validate Email/Password
↓ (valid) / → (invalid: show error)
Check 2FA enabled?
↓ (yes) / → (no: generate session)
Send 2FA code
↓
Verify 2FA code
↓
Generate JWT token & Session
↓
Redirect to Home Feed

3. Post Creation & Publishing Flow

User clicks "Start a post"
↓
Post Editor Opens
↓
User types content / uploads media
↓
Select visibility (Public/Connections/Private)
↓
Add hashtags (optional)
↓
Click "Post"
↓
Content moderation check (spam, abuse)
↓ (pass) / → (fail: reject/flag)
Save to database
↓
Generate feed entries for connections
↓
Trigger notifications
↓
Post appears in feeds

4. Job Search & Application Flow

User navigates to Jobs section
↓
Enter search criteria (title, location, filters)
↓
Search service queries database
↓
Apply ranking algorithm (relevance, date)
↓
Display job listings
↓
User clicks on a job
↓
Show job details page
↓
User clicks "Apply"
↓
Check if Easy Apply available
↓ (yes) / → (no: redirect to company site)
Pre-fill application with profile data
↓
User reviews & submits
↓
Store application record
↓
Notify recruiter
↓
Send confirmation to user

5. Connection Request Flow

User views profile
↓
Clicks "Connect"
↓
Add note (optional)
↓
Send connection request
↓
Store in pending_connections table
↓
Trigger notification to recipient
↓
Recipient receives notification
↓
Recipient accepts/ignores/declines
↓ (accept) / → (ignore/decline: remove request)
Create bidirectional connection
↓
Update both users' networks
↓
Notify requester of acceptance
↓
Update feed algorithms for both users

6. News Feed Generation Flow

User opens LinkedIn app/website
↓
Fetch user's connection IDs
↓
Fetch recent posts from connections
↓
Fetch posts from followed pages/companies
↓
Fetch sponsored content (ads)
↓
Apply ML ranking algorithm
↓
Score based on: engagement, recency, relevance
↓
Mix organic & sponsored content
↓
Cache feed for user
↓
Return paginated feed
↓
Display to user

7. Messaging Flow

User opens messaging
↓
Load conversation list
↓
User selects conversation / starts new
↓
Load message history
↓
Establish WebSocket connection
↓
User types message
↓
Send via WebSocket
↓
Message service receives
↓
Store in messages database
↓
Check recipient online status
↓ (online) / → (offline: send push notification)
Deliver via WebSocket to recipient
↓
Update read receipts
↓
Display in recipient's chat
